# UI/UX Design Spec — medargue Clinical Decision Support

> Source: distilled via the `ui-ux-pro-max` skill for a "clinical decision support dashboard with
> argument-graph visualization, trustworthy/medical, data-dense, explainable AI."
> Target surface: **Streamlit + Graphviz**. This spec is the contract for `ui/app.py` (Phase 4 of any UI card).

## 1. Design direction

- **Pattern:** explainability dashboard — evidence in, reasoning visible, recommendation out.
- **Style:** **Data-Dense Dashboard** — KPI/stat cards, grid layout, minimal padding, maximum data
  visibility. Light + dark both supported. Performance-first, WCAG AA.
- **Tone:** trustworthy, clinical, calm. Color carries meaning (status), never decoration.
- **Avoid:** ornate styling, emoji-as-icons, unfiltered data walls, color used decoratively.

## 2. Color tokens (WCAG-AA tuned)

| Role | Hex | Use |
|------|-----|-----|
| Primary | `#1E40AF` | headers, primary actions, selected agent |
| Secondary | `#3B82F6` | links, secondary accents |
| Accent / CTA | `#D97706` | "Run analysis", key highlights (amber on blue) |
| Background | `#F8FAFC` | app background (light) |
| Foreground | `#1E3A8A` | primary text on light |
| Muted | `#E9EEF6` | panel backgrounds, table stripes |
| Border | `#DBEAFE` | dividers, card borders |
| Destructive | `#DC2626` | high-risk complication, attack edges, refute |
| Ring/Focus | `#1E40AF` | keyboard focus outline |

**Semantic status (complication risk + argument state):**
- Supports/accepted argument → green `#15803D`
- Attacks/refuted argument → red `#DC2626`
- Undecided/snoozed → amber `#D97706`
- Neutral/info → blue `#3B82F6`

Use a **static literal map** for status→class/color (never build color strings at runtime).

## 3. Typography

- **Headings:** `Fira Code` (technical, precise — fits a reasoning tool).
- **Body / data:** `Fira Sans`.
- Import (Streamlit `config.toml` font + injected CSS):
  ```css
  @import url('https://fonts.googleapis.com/css2?family=Fira+Code:wght@400;500;600;700&family=Fira+Sans:wght@300;400;500;600;700&display=swap');
  ```

## 4. Layout (Streamlit)

```
┌────────────────────────────────────────────────────────────┐
│  DISCLAIMER BANNER (sticky, amber/neutral)                   │  ← always visible
├──────────────┬─────────────────────────────────────────────┤
│  SIDEBAR     │  MAIN                                         │
│  - case pick │  Row 1: KPI cards — recommendation, confidence,│
│  - seed      │          #args, #attacks, debate rounds        │
│  - model     │  Row 2: 3 agent panels (History / Diagnostic / │
│  - run btn   │          Treatment) — claims + cited features   │
│              │  Row 3: Argument graph (Graphviz arg-tree)      │
│              │  Row 4: Resolved extension + explanation        │
│              │  Row 5: per-complication risk table             │
└──────────────┴─────────────────────────────────────────────┘
```

- **KPI cards** (`st.columns` + `st.metric`/custom): recommendation label, calibrated confidence,
  argument count, attack count, debate rounds. Minimal padding, equal widths.
- **3 agent panels:** one `st.container` per agent, color-keyed header (each agent a distinct hue
  from the scale). Show the agent's claims, the **feature values it cited**, and what it could NOT
  see (reinforces the partitioning story). Collapsible.
- **Argument graph:** Graphviz digraph — nodes = arguments (colored by accepted/rejected), edges =
  attacks (red, arrowed). Accepted (preferred extension) nodes bordered green/bold. Click/hover →
  argument detail. This is the centerpiece — give it the most space.
- **Explanation:** narrative from the resolved extension, with inline citations to feature values;
  ends with the disclaimer.
- **Risk table:** 12 complications × {predicted, confidence}; row highlight on hover; sortable.

## 5. The disclaimer (mandatory, every screen)

> ⚠ **Research prototype — not clinical advice.** Outputs are generated by a multi-agent argumentation
> system on surrogate data for academic evaluation. Do not use for real clinical decisions.

Sticky top banner + repeated at the foot of the explanation panel. Non-dismissible.

## 6. Interaction & motion

- Hover tooltips (feature meaning from the variable dictionary, argument detail).
- Graph: zoom on click, node highlight on hover.
- Smooth filter/loading transitions (150–300 ms); spinner during debate run.
- Respect `prefers-reduced-motion`.

## 7. Accessibility checklist (pre-delivery, every UI card)

- [ ] Text contrast ≥ 4.5:1 (light + dark)
- [ ] Visible keyboard focus on all interactive elements
- [ ] Icons are SVG (Lucide/Heroicons), never emoji-as-icon (disclaimer ⚠ is the one allowed glyph)
- [ ] `prefers-reduced-motion` respected
- [ ] Responsive at 768 / 1024 / 1440 (Streamlit wide layout)
- [ ] Color is never the ONLY signal — pair with label/icon/shape (accepted vs rejected also bold/border)
- [ ] Graph has a text/table fallback (the risk table + explanation convey the same conclusion)

## 8. Streamlit implementation notes

- `st.set_page_config(layout="wide", page_title="medargue")`.
- Theme via `.streamlit/config.toml` `[theme]` (primaryColor `#1E40AF`, backgroundColor `#F8FAFC`,
  font) + a small injected `<style>` block for fonts/status colors.
- Extract all rendering logic into **pure helper functions** (`ui/components/*.py`) so they unit-test
  without launching Streamlit; `app.py` only wires state → helpers.
- Graph built with the `graphviz` package; status→color via a static dict.
